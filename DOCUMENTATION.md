## 1. Project Overview

**ADM S&OP Dashboard** is a Sales & Operations Planning platform built for **Agua de Madre (ADM)**, a UK-based health drinks company. It consolidates multi-channel sales data, advertising metrics, and inventory levels into a single real-time dashboard, powered by AI-generated strategic recommendations.

### Core Value Propositions

| Capability | Description |
|-----------|-------------|
| **Multi-channel sales visibility** | Unified view of Shopify D2C and Amazon orders |
| **Forecast tracking** | Baseline vs Ambitious plans with accuracy metrics |
| **Ad spend optimization** | Meta and Amazon Ads ROI tracking with budget variance |
| **CAC analytics** | Customer acquisition cost by channel (new vs returning) |
| **AI recommendations** | Daily strategic insights generated by Claude Sonnet |
| **Interactive AI chatbot** | Natural language queries over your business data |
| **Automated data sync** | 4× daily N8N workflows pull fresh data from all sources |
| **Export capabilities** | CSV and PDF export for all dashboard tables |

```
┌─────────────────────────────────────────────────────────────────────┐
│                         Data Sources                                │
│  Shopify Admin API  │  Amazon SP-API  │  Meta Graph API  │  Amazon Ads API  │
└──────────┬──────────┴────────┬────────┴────────┬─────────┴────────┬──────────┘
           │                  │                 │                  │
           └──────────────────┴─────────────────┘──────────────────┘
                                      │
                              ┌───────▼──────────┐
                              │   N8N Workflows   │
                              │  (4 daily crons)  │
                              └───────┬──────────┘
                                      │ upsert
                              ┌───────▼──────────┐
                              │  Supabase (Postgres)│
                              │  13 tables, RLS   │
                              └───────┬──────────┘
                                      │
                    ┌─────────────────┴───────────────────┐
                    │                                     │
             ┌──────▼──────┐                    ┌────────▼────────┐
             │  Next.js 15  │                    │  Anthropic API  │
             │  (App Router)│◄───────────────────│  Claude Sonnet  │
             │  Vercel      │   AI recommendations│  (daily + chat) │
             └──────────────┘                    └─────────────────┘
```

### Request Flow
1. N8N workflows sync data from external APIs into Supabase daily (3–6 AM UTC)
2. The N8N forecast workflow also calls Claude to generate recommendations at 6 AM UTC
3. The Next.js frontend reads from Supabase via server components and API routes
4. Users can trigger on-demand AI recommendations and chat queries from the dashboard

## 5. Database Schema

**Provider:** Supabase (PostgreSQL)  
**Migration file:** `supabase/migrations/001_initial_schema.sql`

### Tables

#### `skus` — Product catalog
```sql
id             UUID PRIMARY KEY
sku_code       VARCHAR(50) UNIQUE NOT NULL
name           VARCHAR(255) NOT NULL
category       VARCHAR(100)          -- drinks | tea | health_products
channel        VARCHAR(50)           -- shopify | amazon | both
is_active      BOOLEAN DEFAULT true
created_at     TIMESTAMP WITH TIME ZONE
updated_at     TIMESTAMP WITH TIME ZONE   -- auto-updated by trigger
```

#### `shopify_orders` — D2C order transactions
```sql
id             SERIAL PRIMARY KEY
order_id       VARCHAR(100) NOT NULL
order_date     DATE NOT NULL
sku_id         UUID REFERENCES skus(id)
quantity       INTEGER
revenue        NUMERIC(10,2)
customer_type  VARCHAR(50)           -- new | returning | subscription
country        VARCHAR(100)
created_at     TIMESTAMP WITH TIME ZONE
UNIQUE(order_id, sku_id)             -- deduplication
```

#### `amazon_orders` — Marketplace order transactions
```sql
id             SERIAL PRIMARY KEY
order_id       VARCHAR(100) NOT NULL
order_date     DATE NOT NULL
sku_id         UUID REFERENCES skus(id)
asin           VARCHAR(20)
quantity       INTEGER
revenue        NUMERIC(10,2)
marketplace    VARCHAR(50)           -- UK | DE | FR | IT | ES
created_at     TIMESTAMP WITH TIME ZONE
UNIQUE(order_id, sku_id)
```

#### `shopify_inventory` — Daily D2C stock snapshots
```sql
id             SERIAL PRIMARY KEY
snapshot_date  DATE NOT NULL
sku_id         UUID REFERENCES skus(id)
location       VARCHAR(255)
available      INTEGER
on_hand        INTEGER
committed      INTEGER
created_at     TIMESTAMP WITH TIME ZONE
UNIQUE(snapshot_date, sku_id, location)
```

#### `amazon_inventory` — FBA stock snapshots
```sql
id             SERIAL PRIMARY KEY
snapshot_date  DATE NOT NULL
sku_id         UUID REFERENCES skus(id)
asin           VARCHAR(20)
fulfillable_quantity    INTEGER
inbound_quantity        INTEGER
reserved_quantity       INTEGER
unfulfillable_quantity  INTEGER
created_at     TIMESTAMP WITH TIME ZONE
UNIQUE(snapshot_date, sku_id)
```

#### `meta_ads` — Meta Ads daily campaign metrics
```sql
id             SERIAL PRIMARY KEY
date           DATE NOT NULL
campaign_id    VARCHAR(100)
campaign_name  VARCHAR(255)
ad_account_id  VARCHAR(100)
impressions    INTEGER
clicks         INTEGER
spend          NUMERIC(10,2)       -- GBP
conversions    INTEGER
revenue        NUMERIC(10,2)
ctr            NUMERIC(8,4)        -- click-through rate
cpc            NUMERIC(10,2)       -- cost per click
created_at     TIMESTAMP WITH TIME ZONE
UNIQUE(date, campaign_id)
```

> **Note:** Meta Ads data is campaign-level only (no SKU attribution).

#### `amazon_ads` — Amazon Ads ASIN-level metrics
```sql
id             SERIAL PRIMARY KEY
date           DATE NOT NULL
sku_id         UUID REFERENCES skus(id)
asin           VARCHAR(20)
impressions    INTEGER
clicks         INTEGER
spend          NUMERIC(10,2)       -- GBP
attributed_sales_14d  NUMERIC(10,2)
orders_14d     INTEGER
acos           NUMERIC(8,4)        -- ad cost of sale
roas           NUMERIC(8,4)        -- return on ad spend
created_at     TIMESTAMP WITH TIME ZONE
UNIQUE(date, asin)
```

#### `forecasts` — Monthly forecast plans per SKU/channel
```sql
id             SERIAL PRIMARY KEY
month          DATE NOT NULL           -- first day of month
sku_id         UUID REFERENCES skus(id)
channel        VARCHAR(50)
baseline_units INTEGER
ambitious_units INTEGER
actual_units   INTEGER
baseline_revenue NUMERIC(10,2)
ambitious_revenue NUMERIC(10,2)
actual_revenue NUMERIC(10,2)
forecast_accuracy NUMERIC(8,4)
generated_at   TIMESTAMP WITH TIME ZONE
UNIQUE(month, sku_id, channel)
```

#### `forecast_parameters` — Per-SKU forecast multipliers
```sql
id             SERIAL PRIMARY KEY
sku_id         UUID REFERENCES skus(id)
channel        VARCHAR(50)
seasonality_index     NUMERIC(6,4)    -- category-specific seasonal multiplier
marketing_uplift      NUMERIC(6,4)    -- ad spend impact factor
price_impact          NUMERIC(6,4)    -- price elasticity factor
channel_effect        NUMERIC(6,4)    -- channel growth factor
last_updated   TIMESTAMP WITH TIME ZONE
```

#### `customer_metrics` — Monthly CAC aggregates
```sql
id             SERIAL PRIMARY KEY
month          DATE NOT NULL
channel        VARCHAR(50)            -- shopify | amazon | total
new_customers  INTEGER
returning_customers INTEGER
subscription_customers INTEGER
cac_new        NUMERIC(10,2)          -- cost to acquire new customer
cac_overall    NUMERIC(10,2)
ltv_estimate   NUMERIC(10,2)
created_at     TIMESTAMP WITH TIME ZONE
UNIQUE(month, channel)
```

#### `production_plans` — Capacity planning
```sql
id             SERIAL PRIMARY KEY
month          DATE NOT NULL
sku_id         UUID REFERENCES skus(id)
planned_units  INTEGER

lead_time_days INTEGER

```

#### `llm_recommendations` — Claude-generated daily insights
```sql
id             SERIAL PRIMARY KEY
month          DATE NOT NULL
generated_at   TIMESTAMP WITH TIME ZONE
run_type       VARCHAR(50)            -- daily | manual
summary_text   JSONB NOT NULL         -- structured JSON (see Section 10)
model_used     VARCHAR(100)
tokens_used    INTEGER
created_at     TIMESTAMP WITH TIME ZONE
```

#### `ambitious_targets` — Top-down investor targets
```sql
id             SERIAL PRIMARY KEY
month          DATE NOT NULL
sku_id         UUID REFERENCES skus(id)
channel        VARCHAR(50)
target_units   INTEGER
target_revenue NUMERIC(10,2)
target_source  VARCHAR(100)           -- investor | board | internal
notes          TEXT
created_at     TIMESTAMP WITH TIME ZONE
UNIQUE(month, sku_id, channel)
```

### Indices (~25 total)
All high-query fields are indexed: `order_date`, `sku_id`, `month`, `date`, `campaign_id`, `asin`, etc.

### Triggers
- `update_updated_at_column()` — automatically updates `updated_at` on the `skus` and `production_plans` tables.

Interactive AI chatbot for data queries.

```json

  "history": [

    { "role": "assistant", "content": "string" }
  ]
```

## 8. N8N Automation Workflows

4 daily cron workflows run in sequence. Import the JSON files from `n8n-workflows/` into your N8N instance.

### Workflow 01 — Shopify Sync (`01-shopify-sync.json`)
**Schedule:** Daily at 3:00 AM UTC  
**Lookback window:** Last 2 days (idempotent)

**Execution:**
1. 3 parallel HTTP requests: orders, products/SKUs, inventory levels
2. JS transform: normalizes line items → `{ sku_code, quantity, revenue, customer_type }`
3. Upsert to `shopify_orders` and `shopify_inventory` (deduped by `order_id + sku_id`)

**Required N8N credentials:**
- Shopify API credential: `SHOPIFY_ACCESS_TOKEN` + `SHOPIFY_STORE_URL`
- Supabase API credential

**Error handling:** Automatic retry up to 3× on HTTP failure.

---

### Workflow 02 — Amazon Sync (`02-amazon-sync.json`)
**Schedule:** Daily at 4:00 AM UTC  
**Lookback window:** Last 2 days

**Execution:**
1. OAuth2 token refresh (client_credentials grant with refresh token)
2. Fetch Amazon SP-API orders + FBA inventory
3. JS transform: extracts `OrderItems` → `{ asin, sku_code, quantity, revenue, marketplace }`
4. Upsert to `amazon_orders` and `amazon_inventory`

**Required N8N credentials:**
- Amazon SP-API: `AMAZON_REFRESH_TOKEN`, `AMAZON_CLIENT_ID`, `AMAZON_CLIENT_SECRET`
- Supabase API credential

---

### Workflow 03 — Ads Sync (`03-ads-sync.json`)
**Schedule:** Daily at 5:00 AM UTC  
**Lookback window:** Last 2 days

**Execution — Meta branch:**
1. Call Meta Graph API v19 `/insights` endpoint at campaign level
2. Fields: `campaign_name`, `impressions`, `clicks`, `spend`, `actions`, `ctr`, `cpc`
3. Upsert to `meta_ads`

**Execution — Amazon Ads branch:**
1. Call Amazon Advertising API v3 report endpoint at ASIN level
2. Metrics: `impressions`, `clicks`, `cost`, `attributedSales14d`, `orders14d`
3. Map ASIN → SKU, upsert to `amazon_ads`

**Required N8N credentials:**
- Meta Marketing API: `META_ACCESS_TOKEN`, `META_AD_ACCOUNT_ID`
- Amazon Ads API: `AMAZON_ADS_CLIENT_ID`, `AMAZON_ADS_CLIENT_SECRET`, `AMAZON_ADS_REFRESH_TOKEN`
- Supabase API credential

### Workflow 04 — Forecast & Recommendations (`04-forecast-recommendations.json`)
**Schedule:** Daily at 6:00 AM UTC (after all data syncs complete)

**Execution:**
1. Fetch active SKUs from Supabase
2. Fetch 6 months of Shopify + Amazon order history
3. Run JS forecasting engine (see Section 9)
4. Insert generated forecasts to `forecasts` table

## 9. Forecasting Engine

The forecasting logic lives in two places — the N8N workflow JS code and the TypeScript/Python implementations.

### TypeScript Implementation
`src/lib/forecast/engine.ts`

### Python CLI Implementation
`adm-sop/scripts/forecasting_engine.py`

```bash
pip install -r adm-sop/scripts/requirements.txt
python3 adm-sop/scripts/forecasting_engine.py --months-ahead 6 --output forecasts.json
```

### Formula

```
Baseline Forecast = 8-week moving average of actual units
                  × seasonality_index
                  × marketing_uplift
                  × price_impact
                  × channel_effect

Ambitious Forecast = Baseline × 1.25 (25% stretch target)
```

### Seasonality Indices (by category)

| Month | Drinks | Tea | Health Products |
|-------|--------|-----|-----------------|
| January | 0.85 | 1.20 | 1.10 |
| February | 0.85 | 1.15 | 1.10 |
| March | 0.95 | 1.00 | 1.05 |
| April | 1.05 | 0.90 | 1.00 |
| May | 1.10 | 0.85 | 0.95 |
| June | 1.15 | 0.80 | 0.90 |
| July | 1.20 | 0.80 | 0.90 |
| August | 1.20 | 0.80 | 0.90 |
| September | 1.10 | 0.90 | 0.95 |
| October | 1.00 | 1.05 | 1.00 |
| November | 0.95 | 1.20 | 1.10 |
| December | 0.90 | 1.25 | 1.15 |

**Note:** `marketing_uplift`, `price_impact`, and `channel_effect` default to `1.0` until ad elasticity data and pricing history are integrated. These are configurable per-SKU in the `forecast_parameters` table.

## 15. Authentication & Security

### Current Status

The current version (0.1.0) is **single-tenant** with no user authentication layer. Access is controlled by keeping the deployment URL private.

### Planned for v1.0

- Supabase Auth (JWT-based)

- All API keys stored in environment variables only
- `SUPABASE_SERVICE_ROLE_KEY` is never sent to the browser
- `ANTHROPIC_API_KEY` is server-side only
- No third-party analytics or tracking scripts
- All external API credentials managed in N8N credential store (not in code)

Per contract (clause 17.i): access to external systems (Google, Meta, Amazon) must use **delegate access only**. No direct password sharing. All credential requests to be sent to `info@virtualcrew.co.uk`.
